# Read This First

Welcome to GPS OS. <img src="../assets/logo/gps-os-mark.svg" width="56" alt="GPS OS">

**GPS OS (GPS Operating System)** is a practical thinking system for transforming operational knowledge into reliable software.

It was created from years of real operational experience, systems thinking, and software development — not as a theoretical framework or a collection of AI prompts, but as a documented way of thinking.

GPS OS exists because understanding a problem is often harder — and more important — than writing code.

Technology changes. Programming languages evolve. AI capabilities continue to improve.

But the ability to understand how work actually happens, design reliable workflows, and build systems that genuinely support people remains timeless.

This repository preserves that way of thinking.

## What GPS OS Is

GPS OS is:

- A practical methodology for designing software from real operational knowledge
- A documented thinking process for product design and system design
- A collection of principles, methods, templates, and real-world examples
- A living body of knowledge that evolves through building real products

## What GPS OS Is Not

GPS OS is **not**:

- A software framework
- A coding tutorial
- An AI prompt collection
- A no-code platform
- A replacement for software engineering

GPS OS complements software engineering by helping teams understand **what should be built before deciding how to build it.**

## Writing Principles

Every document in GPS OS follows the same editorial principles.

1. Never invent stories.
2. Never exaggerate achievements.
3. Every idea must come from real experience.
4. Prefer clarity over clever wording.
5. Explain before convincing.
6. Avoid buzzwords.
7. Write timeless documentation.
8. Preserve authenticity.
9. Keep the author's voice.
10. Document reality.
11. When uncertain, preserve the original thinking instead of improving the writing.
