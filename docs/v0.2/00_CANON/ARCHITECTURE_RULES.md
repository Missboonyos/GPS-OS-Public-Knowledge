# Architecture Rule

GPS OS does not document ideas.

GPS OS discovers systems.

Documentation is the result of discovery, not the objective.

Every concept included in GPS OS must be grounded in operational reality, validated through reasoning, and expressed using the shared language defined by the Canon.

If a new idea does not strengthen the GPS Kernel, it does not belong in GPS OS.

The Canon provides the foundation.

All future models, methods, applications, and AI implementations must remain consistent with this foundation.
